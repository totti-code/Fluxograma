Fluxograma — Troca de Mercadoria (Vencida / Avariada / Perda)

Modelo para supermercado: registro em estoque avaria, decisão de troca com fornecedor, nota de devolução e descarte.

Fluxo Geral
Decisões: “Fornecedor aceita troca?”
flowchart TD A([INÍCIO]) --> B[Produto identificado como problema] B --> C{Qual o tipo?} C -->|Vencido| D[Ir para Fluxo: Produto Vencido] C -->|Avariado| E[Ir para Fluxo: Produto Avariado] C -->|Perda / Lixo| F[Ir para Fluxo: Perda / Lixo] D --> Z([FIM]) E --> Z F --> Z

Você pode editar o texto dentro do bloco flowchart para adaptar ao processo da sua empresa.

Fluxo 1 — Produto Vencido
flowchart TD A([INÍCIO]) --> B[Produto vencido identificado] B --> C[Registrar no sistema / separar em Estoque Avaria] C --> D{Fornecedor aceita troca?} D -->|Sim| E[Emitir Nota de Devolução] E --> F[Enviar relatório / confirmação ao fornecedor] F --> G[Receber produto novo (troca)] G --> H[Lançar entrada (Centro de custo Prod/Prod)] H --> Z([FIM]) D -->|Não| I[Lançar como perda (Centro de custo Troca Vencido)] I --> J[Descartar corretamente conforme política] J --> Z
Fluxo 2 — Produto Avariado
flowchart TD A([INÍCIO]) --> B[Produto avariado identificado] B --> C[Registrar no sistema / separar em Estoque Avaria] C --> D{Fornecedor aceita troca?} D -->|Sim| E[Emitir Nota de Devolução] E --> F[Enviar relatório / confirmação ao fornecedor] F --> G[Receber produto novo (troca)] G --> H[Lançar entrada (Centro de custo Prod/Prod)] H --> Z([FIM]) D -->|Não| I[Lançar em Centro de custo Troca Avariado] I --> J[Descartar corretamente conforme política] J --> Z
Fluxo 3 — Perda / Lixo
flowchart TD A([INÍCIO]) --> B[Identificação da perda / lixo] B --> C[Registrar motivo (ex.: vazamento, contaminação, queda, aberto)] C --> D[Lançar no centro de custo Perda/Lixo] D --> E[Descartar corretamente conforme política (coleta, segregação, etc.)] E --> Z([FIM])
